# 감성적 설계 — 심리·생리 근거

알림 시점·문구·기본값이 **왜 그렇게 정해졌는지**의 근거를 기록한다. 상위 원칙은 [design-principles.md](./design-principles.md).

## 1. 절정-대미 법칙 (Peak-End Rule)

경험의 기억은 **절정 순간과 마지막 순간**이 지배한다.

- **적용**: 하루의 '마지막 순간'인 **정시 퇴근 알림에 오늘 실근무 요약**을 담아 긍정적으로 마무리한다("오늘 실근무 N시간 M분 — 수고하셨습니다!"). 일일 마감 요약(`summary.log`)도 같은 맥락.

## 2. 서카디안 리듬 (하루 주기)

각성도는 아침 상승 → 점심 무렵 피크 → 오후 초 저하(post-lunch dip) → 저녁 하강 → 밤 저각성(수면 준비)으로 움직인다.

- **적용**: **수면시간 기본값 00:00–07:00** — 저각성·수면 준비 시간대에는 어떤 알림도 깨우지 않는다(전체 무음). **야간 시간대 기본값 22:00–06:00** — 저각성 시간대의 근무를 야간근무로 집계·경고한다.

## 3. 블루라이트와 밤

밤의 화면 자극(특히 파랑 파장)은 멜라토닌 분비를 억제해 수면을 방해한다.

- **적용**: 수면시간 무음의 생리적 근거. 야간 알림 문구는 자극적 경고 대신 **차분한 권유**("무리하지 마세요")로 각성을 더 끌어올리지 않는다.

## 4. 울트라디안 리듬 (약 90분 집중 주기)

집중력은 약 90분 주기로 오르내린다 — 휴식 없는 장시간 연속 작업은 효율과 건강을 모두 해친다.

- **적용**: **연속근무 한도(기본 60분)** 초과 시 휴식 권유 알림. 이후에도 1시간 간격으로만 절제해 반복하고, 휴식하면 리셋된다.

## 5. 색채 심리

한색(파랑)은 차분·신뢰·집중을, 난색은 각성·경고를 연상시킨다. 채도·명도가 높을수록 각성이 올라간다.

- **적용**: 히트맵은 회색(0) → 옅은 파랑 → 진한 파랑의 **단일 색상 스케일** — 이 앱은 차분한 자기 관찰 도구이지 경쟁·압박 도구가 아니다. 주간 리포트의 요일 막대도 같은 파랑 계열이며, **표준을 넘은 부분만 저채도 난색**으로 구분한다(빨강 경고가 아닌 담백한 구분 — 죄책감 유발 금지). 트레이 아이콘은 파랑(측정 중)/회색(일시정지). 경고성 알림(야간·초과·휴식)만 Warning 아이콘을 쓴다.
- **접근성**: 색만으로 의미를 전달하지 않는다 — 히트맵은 행 합계 숫자를 병기하고, 아이콘 상태는 툴팁 텍스트로도 알 수 있다.

## 6. 적용 현황 (원리 → 구현)

| 원리 | 적용 (기본값·기능) | 구현 위치 (`src/Program.cs`) |
|---|---|---|
| 절정-대미 (§1) | 정시 퇴근 알림의 오늘 실근무 요약 · 일일 마감 요약 | `TrayApp.CheckNotifications` · `FinalizeDay` |
| 서카디안 (§2) | 수면시간 00:00–07:00 전체 무음 · 야간 22:00–06:00 집계/경고 | `Config` 기본값 · `Notify()` 수면 게이트 · `InWindow` |
| 블루라이트 (§3) | 야간 문구를 차분한 권유로 | `CheckNotifications` 야간 분기 |
| 울트라디안 (§4) | 연속근무 한도 60분 → 휴식 권유(1시간 간격, 휴식 시 리셋) | `CheckNotifications` · `AccountAway` |
| 색채 심리 (§5) | 히트맵 단일 파랑 sqrt 스케일 · 주간 리포트 막대 파랑 + 표준 초과분만 저채도 주황 · 아이콘 파랑/회색 · Warning 아이콘은 경고성 알림만 | `HeatmapForm.ColorFor` · `WeeklyForm.OnPaint` · `BuildIcons` |

기본값을 바꾸거나 새 기능을 넣을 때 이 표의 근거가 유지되는지 본다. 근거가 달라지면 이 문서도 같은 커밋에서 고친다.

## 7. 새 알림 체크리스트

- [ ] `Notify()` 를 경유해 수면시간·마스터 토글 게이트를 우회하지 않는가
- [ ] 문구가 권유형이고, 시간대의 각성 상태를 불필요하게 끌어올리지 않는가(§2·§3)
- [ ] 반복이 절제됐는가 — 1시간 버킷 카운터, 시작·자정·설정변경 시 prime 으로 몰림 방지
- [ ] 하루의 마무리 경험(퇴근 이후)을 해치지 않는가(§1)
- [ ] 아이콘(Info/Warning)이 전하려는 감정 강도와 일치하는가
